**SQL OIL MARKTEING COMPANY (OMC) DATA WAREHOUSE****#  1. PROJECT OBJECTIVE*** Design, implement and run a PostgreSQL data warehouse pipeline for the operations of an oil marketing company (OMC) with more than 90 managed stations.**## 2. ARCHITECTURE**This data warehouse applies a medallion architecture. In this framework, the following are the key workflow layers:* Bronze layer    → ingestion of raw data* Silver Layer     → cleaning / transformation of raw data* Gold Layer      → transformation of data into business-ready schema  and development of information for business analytics* Analytics layer → development of views and analytical data for management analysis and reporting&#x20;**## 3. DATA MODEL**Fact tables:* sales* stocks* prod\_topp* var* rttDimension tables:* dates\_tab* stations* areas**## 4. ERD**&#x20;   **YET TO BE INCLUDED** **## 5. ELT WORKFLOW**1. INGESTION: Load csv files (raw data) into bronze layer2. CLEANING: Merge, clean and standardize records in silver layer to create master file3. TRANSFORMATIONS: Create dimensional and fact tables in gold layer from master file4. ANALYTICS: Create analytics files and complete data analysis in analytics layer**## 6. SQL FEATURES USED*** Common Table Expression (CTEs)* Constraints, primary and foreign keys* Indexes* Joins* Stored Procedures* Triggers* Views* Windows functions**## 7. PERFORMANCE OPTIMIZATION*** Indexed foreign keys* Reduced query execution time using partitions (\*partition annual tables for 2024 and 2025)* Optimized joins with partitioned indexes**## 8. ANALYTICS: KEY PERFORMANCE INDICATORS*** Daily station performance reports: top sales, variance analysis, sales per stock;* Fraud detection reports: variance analysis* Stock trends* RTT performance tracking* Risk analysis**## 9.
